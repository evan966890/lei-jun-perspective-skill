# 雷军视角 skill 的萃取过程

## 1. 目标不是“模仿说话”，而是提炼可复用的思维结构

一开始就把目标定得很明确：

不是做一个只会说“Are you OK”式梗化模仿包，
而是做一个在 agent 场景里真正可用的技能。

所以萃取目标分成四层：

1. **人物画像**：雷军稳定的人物底色是什么
2. **决策方法**：他如何判断行业、产品、组织问题
3. **表达风格**：他如何把复杂问题讲给大众听
4. **应用 prompt**：如何让 agent 在不同场景下复用这些能力

## 2. 先建 dossier，再蒸馏

先在本地搭建了：

`research/lei-jun-dossier/`

它不是一个单文件笔记，而是一个分层资料库，按如下类别归档：

- `01-weibo/`：微博相关抓取
- `03-launches/`：年度演讲与发布会资料
- `04-interviews/`：深度访谈与访谈整理稿
- `05-video/`：视频页面、字幕、评论等
- `07-company/`：官方公司与 IR 资料
- `10-distill/`：蒸馏产物

这样做的目的是先把“证据层”压实，再做 skill 层封装。

## 3. 本次 v0.2 主要使用了哪些资料

### A. 官方资料
用于确认人物基础身份、角色和公开官方表述。

主要包括：
- 小米官网 founder / 管理团队资料
- Xiaomi IR 的 Lei Jun profile

这部分主要解决“事实锚点”问题，避免人物基础信息漂移。

### B. 年度演讲材料
本次优先用了：
- 2020
- 2022
- 2024

原因很直接：
- 这三年材料可读性较强
- 有比较稳定的公开表达主线
- 能看见雷军如何讲产品、组织、挫折、信念与用户

尤其：
- 2020 更像“创业与小米方法”回看
- 2022 更像“低谷、反思、信念”表达
- 2024 更像“造车、勇气、组织、产品定义”表达

### C. 深度访谈材料
重点使用了李翔相关长访谈整理材料。

这类材料价值非常高，因为它不像发布会那样只讲“输出后的结论”，
而是更接近：
- 为什么这样判断
- 为什么这样定价
- 为什么这么组织团队
- 如何看营销、用户、品牌、组织协同

这部分基本构成了 v0 决策系统的主骨架。

### D. 近期微博样本
已抓到 200 条近期微博样本。

虽然不是全量，但足够用于观察：
- 最近说话节奏
- 最近关注主题
- 微博场域下的短表达风格
- 推荐式、技术帖式、直播导流式口吻

### E. 本机长文本书籍输入
在 `~/Desktop/leijun/` 与本地 dossier 中确认存在一批雷军 / 小米相关长文本书籍。
经去重后，当前按 **9 本唯一文本** 参与内部蒸馏，用来补足：
- 早年经历
- 创业阶段叙事
- 长期价值观线索
- 反复出现的心智模型与口头禅

这些文本 **只用于内部蒸馏**，不会原样进入公开仓库分发。

## 4. 为什么没有直接使用整批书籍全文

资料库里存在一批用户侧导入文本，但它们已经被标记为：
- `private_unverified`
- `potentially_copyright_restricted`

因此本次发布的公开版 skill **没有把这些全文作为主证据**。

这样做有两个原因：

1. **合规边界**
   公开市场版本不能把边界不清的输入当成核心材料。

2. **方法边界**
   这版 skill 要能向外说明“我是怎么来的”，就必须优先依赖公开、可说明来源的材料。

## 5. 证据权重是怎么分的

v0 里采用的是一个简单但很实用的证据优先级：

### 第一层：官方一手资料
- 官网
- IR
- 官方公开表达

### 第二层：雷军本人长篇表达
- 年度演讲
- 深度访谈
- 视频字幕
- 近期微博样本

### 第三层：第三方整理稿
- 用于辅助归纳
- 不单独决定结论

### 第四层：边界不清或高风险输入
- 暂不进入公开版主结论

## 6. 真正萃取时，抽的不是“名句”，而是四类结构

### 6.1 人物底色
最终压出来的稳定结论是：

**工程师底色，产品经理方法，创业者意志，强公众表达能力。**

### 6.2 决策结构
从访谈和演讲里能反复看到：
- 终局思维
- 赢家通吃
- 危机驱动 + 战略驱动 + 个人意志
- 先守正再出奇
- 高压强投入
- 组织共识优先

### 6.3 表达结构
雷军真正特别的地方，不只是讲得好，而是：
- 白话
- 高密度
- 高结构化
- 能传播
- 能把复杂问题压成几句普通人记得住的话

### 6.4 可复用场景
最后把 skill 的用途压到了几个最容易落地的场景：
- 演讲稿
- 采访回答
- 发布会表达
- 微博文案
- 战略/组织/产品分析

## 7. 为什么先发 v0，再做 v0.2

因为当时微博全量抓取被验证码卡住了。

如果等全部完美再做，这个 skill 会一直停留在“准备中”。
所以策略是：

**先发一版边界清楚、证据可说明、已经可用的 v0。**

随后再补一个 `v0.2`：
- 纳入本机长文本书籍输入做内部蒸馏补强
- 用 nuwa-style 的 5 层结构重新整理输出
- 继续保留公开版和内部证据层之间的边界

这也是整个项目最重要的取舍之一。

## 8. 最后如何落成可发布 skill

在 `10-distill/` 里先生成了：
- 核心人物画像
- 表达风格
- 语录锚点
- source policy
- application prompts

然后再把真正需要给 agent 用的部分收敛成：
- `SKILL.md`
- `references/core-portrait.md`
- `references/voice-style.md`
- `references/quote-bank.md`
- `references/application-prompts.md`

也就是说：

- **dossier** 负责“研究”
- **skill** 负责“调用”

## 9. 这版 skill 的边界

它已经足够支持：
- 雷军式表达
- 小米式方法论分析
- 创始人视角写作

但还没到“最终完整版”。

后续仍值得继续补：
- 2021 / 2023 演讲结构化
- 更长时间跨度微博样本
- 更多视频原话对齐
- 更细的场景化 prompt 模板

## 10. 一句话总结

这个 skill 的核心不是“像雷军说话”，而是：

**让 agent 能更接近雷军那套“看问题、做判断、讲清楚”的方式。**
