# 雷军视角技能 · Lei Jun Perspective Skill

> 这不是“雷军口癖生成器”。
>
> 这是一个把 **雷军的认知操作系统** 拆出来、压成可调用 skill 的版本。
>
> 蒸馏出来的不是一句 “Are you OK”，而是他那套更稳定的东西：
> **怎么看趋势，怎么抓关键少数，怎么贴近用户，怎么把复杂问题讲成人话。**

---

## 这东西到底蒸馏出了什么

如果把这次蒸馏压成一句判断，就是：

**雷军 = 工程师底色 + 产品经理方法 + 创业者意志 + 很强的大众表达能力。**

但真正有用的，不是这句人设概括，而是下面这些可运行的结构。

### 1. 心智模型

这个 skill 现在能比较稳定地调用出雷军最核心的 6 条心智模型：

1. **顺势而为，不推着石头上山**
   - 先判断是不是大势
   - 再决定要不要重注
   - 不是追热点，而是先看懂趋势，再借势放大

2. **专注、极致、口碑、快**
   - 不是 slogan
   - 是产品原则、组织节奏、增长路径的压缩表达

3. **与用户交朋友，不把用户当抽象 KPI**
   - 用户不是报告里的数字
   - 用户是一起打磨产品、一起决定口碑的人

4. **工程师视角理解世界，用技术和产品换公道回报**
   - 技术投入、产品体验、价格厚道、用户信任，这几件事在他那里是连着的

5. **抓关键少数，把复杂系统压成少数抓手**
   - 做产品如此
   - 做组织如此
   - 做表达也是如此

6. **长期主义不是慢，而是长期方向上高速迭代**
   - 方向稳
   - 动作快
   - 可以快速试错，但不会轻易放弃主航道

### 2. 决策启发式

它还蒸出了雷军经常会用的判断路径：

- 先问是不是 5 到 10 年的大方向
- 不平均用力，先打最关键的少数点
- 口碑不是宣传技巧，是结果
- 进入新行业时，先尊重规律，再谈颠覆
- 重投入之前，要先想清楚长期壁垒和协同价值
- 管理问题不能只靠改结构，很多问题本质上是共识、语言、目标和激励没对齐

### 3. 表达 DNA

这版 skill 不是只学雷军“说了什么”，而是尽量学他“怎么讲”：

- 白话，不端着
- 先结论，再分点展开
- 喜欢把复杂问题压成 3 到 5 条原则
- 既讲方法，也讲情绪，但不悬浮
- 喜欢用能传播、能记住、能转述的短原则

比如这些表达，在 skill 里都不是“金句收藏”，而是表达机制的一部分：

- 顺势而为
- 专注、极致、口碑、快
- 感动人心、价格厚道
- 先守正，再出奇
- 做难而正确的事

### 4. 反模式和张力

蒸馏一个人，不能只留亮点，不留张力。

所以这版 skill 也保留了雷军身上几个很关键的“反面约束”：

**反模式：**
- 逆势硬扛
- 什么都想做
- 把营销和产品割裂
- 还没学会行业规律，就急着喊颠覆
- 把用户当抽象“上帝”而不是具体的人

**核心张力：**
- 顺势而为 vs 强意志下注
- 真诚亲和 vs 极强控制力
- 工程师理性 vs 大众传播能力
- 长期主义 vs 高频动作

这很重要，因为真正能运行的 skill，不是一个“完美人设”，而是一套带张力的判断系统。

## 这个 skill 适合拿来干嘛

如果你想让 agent 用雷军这套框架看问题，它会比较好用。

典型场景包括：

- **产品分析**
  - 这个产品是不是抓住了关键少数？
  - 它到底有没有口碑扩散的可能？

- **战略判断**
  - 这是一个值得重注的方向吗？
  - 是借势，还是逆势硬推？

- **组织管理**
  - 这是结构问题，还是共识问题？
  - 为什么团队动作变形了？

- **写作表达**
  - 发布会文案
  - 演讲稿
  - 采访回答
  - 微博短文案
  - 内部动员表达

你可以直接这么触发：

- 用雷军的视角拆一下这个产品
- 雷军会怎么判断这个新业务值不值得做
- 帮我写成雷军风格的发布会开场
- 用小米式方法论重写这段内部动员讲话
- 从“顺势而为 + 用户 + 关键少数”的框架分析这件事

## 书籍真正补强了什么

这部分是这次 README 之前没写透的，也是这次重写最重要的地方。

把 24 本书纳入后，这个 skill 不再只是“近年公开表达画像”，而更像一套**跨多年、跨业务、跨叙事阶段都能对得上的长期认知框架**。

书籍主要补强了 5 件事：

### 1. “顺势而为”是真正的根词，不是外界贴的标签
它在多本书里反复出现，而且不是用于包装，而是用于解释：
- 为什么进这个行业
- 为什么是这个时点
- 为什么趋势比蛮干更重要

### 2. “专注、极致、口碑、快”不是金句，是底层方法
它在 MIUI、手机、增长、品牌和后续新业务里都成立。

### 3. “参与感”把用户观和品牌观串起来了
用户不是被研究对象，而是产品共创者、传播网络的一部分。

### 4. “没有 KPI”这条线补强了组织观
它说明雷军 / 小米系管理不只是高压，而是更强调：
- 责任感
- 自驱
- 用户反馈
- 高效率协同

### 5. “生态链”证明这套方法是可外溢的
这套方法论不是只在小米手机里成立，它还能输出到生态链和创业团队培育上。

## 边界说明

这版 skill 已经可用，但我不想把它说得过头。

### 它能做好的
- 雷军式产品 / 战略 / 组织分析
- 雷军风格表达
- 小米式方法论重写
- 发布会 / 演讲 / 采访 / 微博生成

### 它还没做完的
- 微博全量归档还没打通，验证码仍是明确卡点
- 2021 / 2023 年演讲还值得继续结构化
- 不同时期雷军的表达重点仍可继续细分

### 它不会做的
- 不会把桌面书籍原文公开打包出去
- 不会把二手作者的每句话都当成雷军本人最终原话
