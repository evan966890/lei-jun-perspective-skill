# 雷军视角技能 · Lei Jun Perspective Skill

一个面向 OpenClaw / AgentSkill 场景的 **雷军方法论与表达风格技能**。

它不是“模仿秀”，而是基于 **公开年度演讲、深度访谈、官方资料、近期微博样本，以及本机长文本书籍输入的内部蒸馏结果** 做出的结构化萃取，目标是让 agent 在需要时，能够用更接近雷军的方式去：

- 看产品
- 看战略
- 看组织
- 看用户
- 写演讲 / 发布会 / 采访 / 微博风格文案

## 这个 skill 能做什么

激活后，它更擅长：

- 用 **工程师 + 产品经理 + 创业者** 的混合视角分析问题
- 先讲结论，再分 3 到 5 点展开
- 把复杂商业问题压缩成大众能理解的原则
- 强调用户、长期主义、组织执行、尊重规律、做难而正确的事
- 生成更接近雷军风格的：
  - 演讲稿
  - 采访回答
  - 发布会口径
  - 微博短文案
  - 内部动员表达

## 不适合做什么

- 历史细节考据
- 闭门内部信息判断
- 法务 / 财务 / 监管类高专业最终结论
- 把雷军简化成“纯营销之神”式人设模仿

## 中文详细说明

这个 skill 的核心不是“把雷军说话腔调抄过来”，而是把更稳定、可迁移的部分抽出来：

1. **决策系统**
   - 先判断终局，再决定是否入场
   - 先做深度调研，再高压强投入
   - 先守正，再出奇
   - 强调组织共识和执行系统

2. **用户观**
   - 用户不是抽象名词，而是真实场景中的具体人
   - 100 份报告，不如一个真实用户面对面的沟通
   - 产品、传播、品牌，最终都要回到用户洞察

3. **表达风格**
   - 口语化
   - 先结论后解释
   - 喜欢用 3 到 5 条原则组织表达
   - 能讲情绪，但不悬浮
   - 会用比喻和短句，把复杂问题讲简单

4. **价值观**
   - 长期主义
   - 真诚
   - 勇气
   - 尊重规律
   - 做难而正确的事
   - 人生值得被奖励

## 萃取过程

我把萃取过程单独写在：

- [`docs/extraction-process.md`](docs/extraction-process.md)

这里先给简版：

### 第 1 步：先归档，再蒸馏
先搭了 `research/lei-jun-dossier/`，把雷军相关公开材料按类别沉淀下来，而不是直接凭印象写 skill。

### 第 2 步：给证据分层
本次优先使用：

- **官方资料**：小米官网 / IR
- **年度演讲**：2020、2022、2024
- **深度访谈**：李翔相关长访谈整理与公开材料
- **近期微博样本**：已抓到的 200 条样本
- **长文本书籍输入**：本机桌面与 dossier 中去重后的 9 本雷军 / 小米相关文本，仅用于内部蒸馏补强，不随公开仓库分发

### 第 3 步：排除高风险输入
没有把用户侧导入、且被标记为 `potentially_copyright_restricted` 的书籍全文作为主证据。

### 第 4 步：先做 v0
在微博全量尚未补齐的情况下，先做一版 **可用但不过度宣称“全量”的 v0**。

### 第 5 步：拆成可复用模块
最终没有只写一个 prompt，而是拆成：
- 人物画像
- 表达风格
- 语录锚点
- 应用 prompts

这样 agent 可以按需读取，不会把所有信息糊成一团。
